Create new collector for data retrieving

An adaptor is an .jar package that is plugged in to the collector to query data. Followings are some guidelines and notations to create this:
  1. Create a normal Java project, then a class which extends the UnitInstanceCollector interface. The class should implement:
    • The mechanism to collect the information
    • The mechanism to identify the instance.
  2. The application-specific information must be marshalled into string-based format (e.g. JSON, XML) in order to store in the UnitInstance object.
  3. Developer can reuse the function "readAdaptorConfig(String parameter)" to read the value in adaptor.conf going with the adaptor.
  4. SALSA will merge the collected information with existed instances in the database, thus the identification must enable SALSA to link with existed information.
The artifact of the adaptor will be plugged in the collector as the previous guide. Please check the adaptor written for GovOps service. Other available adaptors are here.
